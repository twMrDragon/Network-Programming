# week0601 核心編譯

## 可能會遇到的問題

- 編譯時出現`the frame size of 1040 bytes is larger than 1024 bytes`
  重雨說這是 WARNING，所以他無視掉了
- 東西很大<br>
  在編譯完成後我的 linux-6.4.9 資料夾大小為 21.5G，請在編譯前看你的容量夠不夠
- 時間很長<br>
  我用筆電總共編譯了 2 小時 51 分鐘，可以放著他去做其他事；或是開啟電腦不休眠後跑去睡覺

## 步驟

```bash
# 更新軟體包(非必要
sudo apt update
sudo apt upgrade

# 安裝編譯所需套件
sudo apt install wget build-essential bison flex libncurses-dev libssl-dev libelf-dev
sudo apt-get install dwarves
sudo apt-get install zstd

# 下載新版 Kernel 程式碼
# 使用wget下載
wget https://cdn.kernel.org/pub/linux/kernel/v6.x/linux-6.4.9.tar.xz
# 解壓縮
tar xavf linux-6.4.9.tar.xz
# 切換目錄
cd linux-6.4.9

# 設定 config
# 不太需要改甚麼，save、exit就好
make menuconfig
scripts/config --disable SYSTEM_TRUSTED_KEYS
scripts/config --disable SYSTEM_REVOCATION_KEYS

# 建立新的資料夾並切換到該資料夾中
mkdir hello
cd hello

# 新增一個 System call 程式
gedit hello.c
```

```c
// hello.c
#include <linux/kernel.h>
#include <linux/syscalls.h>
SYSCALL_DEFINE0(hello){
    printk("Hello World.\n");
    return 0;
}
```

```bash
# 新增 Makefile，確保客製的程式被編譯進 Kernal
gedit Makefile
# 內容放
obj-y:=hello.o

# 切回 Kernel 根目錄，修改 Kbuild
cd ..
gedit Kbuild

# 找到這兩行
obj-y += virt/
obj-y += $(ARCH_DRIVERS)

# 再後面插入
obj-y += hello/

# 新增客製 System Call 到 System Call Table
gedit arch/x86/entry/syscalls/syscall_64.tbl

# 在 common 最後新增一行，此類 system call 到 450，此填 451
451 common hello sys_hello

# 修改 System Call Header
gedit include/linux/syscalls.h

# 在asmlinkage long sys_ni_syscall(void);後新增
asmlinkage long sys_hello(void);

# 設定 Kernel
make menuconfig

# 編譯新版 Kernel
sudo make -j4 clean
# 編譯新版 Kernel並安裝
sudo make -j4
sudo make modules_install -j4
sudo make install -j4

# 之後重開機
# 重開機後檢查kernel是否改變
uname -a

# 測試新增的System Call
# 在Home目錄寫C程式執行System Call (451為syscall 編號)
gedit test.c
```

```c
// test.c
#include <linux/kernel.h>
#include <sys/syscall.h>
#include <sys/types.h>
#include <stdio.h>
#include <unistd.h>
int main(){
  int a = syscall(451);
    printf("Hello World %d\n", a);
    return 0;
}
```

```bash
# 編譯程式
gcc -g -Wall test.c -o test
# 執行程式
./test
# 查看kernel輸出的訊息
dmesg | grep Hello
```
